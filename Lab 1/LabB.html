Preguntas a Responder

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • El internet es un sistema que conforma una gran red de comunicación mundial, a través de la cual se mantienen interconectadas millones de personas por medio de diversos servicios que ofrecen la conexión. La web o World Wide Web es un sistema que funciona a través de internet, por el cual se pueden transmitir diversos tipos de datos a través del Protocolo de Transferencia de Hipertextos o HTTP, que son los enlaces de las páginas web.

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • Usa GET para extraer información solamente
    Usa POST para crear nuevos recursos y para mandar informacón al servidor
    Usa PUT para actualizar un recurso existente
    DELETE elimina recursos
    HEAD para extraer información pero de manera parcial (encabezados y líneas de estado)
    PATCH permite hacer una modificación parcial al recurso

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • POST porque estas ingresando información al servidor

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • GET porque estas cargando la información desde el servidor

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • La solicitud ha tenido éxito. El significado de un éxito varía dependiendo del método HTTP:
    GET: El recurso se ha obtenido y se transmite en el cuerpo del mensaje.
    HEAD: Los encabezados de entidad están en el cuerpo del mensaje.
    PUT o POST: El recurso que describe el resultado de la acción se transmite en el cuerpo del mensaje.
    TRACE: El cuerpo del mensaje contiene el mensaje de solicitud recibido por el servidor.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • El servidor no pudo encontrar el contenido solicitado. Este código de respuesta es uno de los más famosos dada su alta ocurrencia en la web.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • El servidor ha encontrado una situación que no sabe como manejarla, por lo tanto es su responsabilidad encontrar el error y corregirlo.

    • Es obsoleto como: acronym, applet, basefont, bgsound. big. blink, center, dir, font, frame, frameset, isindex, listing, marquee, nobr, noembed, noframes, plaintext, s, spacer, strike, tt, u, wbr, xmp.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • No es necesario el cierre de las etiquetas img, br, hr, input, etc. Se puede usar tanto minúsculas como mayúsculas en el código a diferencia del XHTML que solo admite minúsculas. El Doctype o declaración del documento está completamente minimizado, en HTML5 solo es necesario emplear: Introduce nuevos marcadores para sumarlos a los existentes div en usos específicos, por ejemplo: nav, footer, audio, video, etc.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    • < table > te permite hacer la tabla
    < tr > crea las filas o rows
    < td > permite crear columnas o data
    < thead > es para el encabezado
    < tbody > es para el cuerpo
    Puedes usar < colspan > y < rowspan > para fucionar columnas y filas

  • ¿Cuáles son los principales controles de una forma HTML5?

    • < input > permite hacer entradas al formulario
    Se puede modificar el tipo de entrada a texto, boton, checkbox, entre otros
    < action > es la acción que sucede cuando subes el formulario
    < target > es la página que puede redireccionar al subir el formulario
    < method > especifica el metodo HTTP al subir el formulario
